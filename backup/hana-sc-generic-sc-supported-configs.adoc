= Obtenga información sobre las configuraciones compatibles con SnapCenter para SAP HANA
:allow-uri-read: 


[role="lead"]
SnapCenter admite una amplia gama de arquitecturas de sistemas SAP HANA y escenarios de implementación en plataformas de almacenamiento locales y en la nube. Obtenga información sobre las configuraciones de SAP HANA compatibles, las combinaciones de plataformas, los protocolos de almacenamiento y las operaciones de respaldo y restauración disponibles para cada entorno.



== Configuraciones de SAP HANA compatibles

SnapCenter admite las siguientes configuraciones y funciones de HANA:

* Sistemas de host único SAP HANA
* Sistemas host múltiples de SAP HANA
+
** Requiere una implementación de complemento central como se describe en link:hana-sc-generic-concepts.html#deployment-options-for-snapcenter-plug-in-for-sap-hana["Opciones de implementación del complemento SnapCenter para SAP HANA"].


* Sistemas SAP HANA MDC
+
** con uno o varios inquilinos


* Sistemas SAP HANA con múltiples particiones
* Replicación de sistemas SAP HANA
* Cifrado de SAP HANA (datos, registro, copia de seguridad)




== Configuraciones de plataforma e infraestructura compatibles

SnapCenter admite las siguientes combinaciones de plataformas de host, sistemas de archivos y plataformas de almacenamiento.

[cols="24%,47%,29%"]
|===
| Plataforma de alojamiento | Conexión de almacenamiento y sistema de archivos de SAP HANA | Plataforma de almacenamiento 


| VMware | Montajes NFS en el invitado | ONTAP AFF 


| VMware | Almacén de datos FC con VMFS + VM con XFS con o sin Linux LVM | ONTAP AFF o ASA 


| KVM | Montajes NFS en el invitado | ONTAP AFF 


| Servidor físico | Montajes NFS | ONTAP AFF 


| Servidor físico | FC SAN + y XFS con o sin Linux LVM | ONTAP AFF o ASA (*) 


| Máquina virtual de Azure | Montajes NFS | Azure NetApp Files 


| AWS EC2 | Montajes NFS | FSx para ONTAP 
|===
(*): Soporte ASA disponible a partir de la versión 6.2 de SnapCenter


NOTE: Los complementos de HANA y Linux solo están disponibles para la plataforma de CPU Intel. Para Linux en IBM Power, se debe configurar una implementación de complemento HANA central como se describe en link:hana-sc-generic-concepts.html#deployment-options-for-snapcenter-plug-in-for-sap-hana["Opciones de implementación del complemento SnapCenter para SAP HANA"].



== Funciones y operaciones compatibles



=== Explicación de la abreviatura

* VBSR: SnapRestore basado en volumen + Un SnapRestore basado en volumen revierte el volumen al estado de la instantánea.
* SFSR: SnapRestore de un solo archivo + Se puede usar un SnapRestore de un solo archivo para restaurar archivos o LUN específicos dentro de un volumen.


Véase también https://docs.netapp.com/us-en/snapcenter/protect-hana/reference_types_of_restore_performed_for_auto_discovered_sap_hana_databases.html["Tipos de operaciones de restauración para bases de datos SAP HANA detectadas automáticamente"]
